# アーキテクチャ

## システム全体図

```
┌─────────────────────────────────────────────────────────────────┐
│                    AWS CloudWatch                                │
│              (実行ログ、メトリクス監視)                         │
└─────────────────────────────────────────────────────────────────┘
                         ▲
                         │ ログ送信
                         │
┌─────────────────────────────────────────────────────────────────┐
│                    EventBridge                                   │
│              (1時間ごとにLambda関数を実行)                      │
└────────────────────────────┬────────────────────────────────────┘
                             │ RunTask命令
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│              AWS Lambda (Python)                                │

└────────────────────────────┬────────────────────────────────────┘
                             │
                             ▼

│         AWS ECS Fargate                                          │

                ▼            ▼            ▼

```

## コンポーネント説明

### 1. EventBridge
- **役割**: 定期スケジュール実行
- **スケジュール**: 1時間ごと（Cron: `0 * * * ? *`）
- **動作**: Lambda関数を呼び出し

### 2. Lambda関数（Python）
- **役割**: Fargateタスク起動
- **動作**: `RunTask`コマンドを実行してすぐに終了
- **タイムアウト**: 60秒
- **メモリ**: 128MB

### 3. AWS ECS Fargate
- **役割**: Playwrightコンテナを実行
- **CPU**: 0.5 vCPU（最小構成）
- **メモリ**: 1GB
- **プラットフォーム**: Fargate

### 4. Playwrightアプリケーション（Node.js）
- **役割**: Webスクレイピング実行
- **処理内容**:
  1. ブラウザ起動
  2. https://www.yahoo.co.jp/ にアクセス
  3. title タグを取得
  4. S3に保存
  5. ブラウザクローズ

### 5. S3（PlaywrightOutput）
- **役割**: スクレイプ結果保存
- **キー構造**: `{サイト名}/{ファイル名}.txt`
- **例**: `yahoo/title_1704067200000.txt`

## データフロー

```
1. EventBridge (毎時0分)
   ↓ スケジュール実行
2. Lambda関数
   ↓ ECS RunTask実行
3. Fargate コンテナ起動
   ├─ Playwrightアプリ起動
   ├─ ブラウザセッション開始
   ├─ https://www.yahoo.co.jp/ アクセス
   ├─ title 取得
   ├─ S3へ保存
   └─ コンテナ終了
4. CloudWatch Logs
   ↓ ログ記録
5. S3 PlaywrightOutput
   └─ 結果保存完了
```

## セキュリティ考慮事項

### IAM権限
- **Playwright-Role**: ECS実行用ロール
  - S3読取・書込権限
  - CloudWatch Logs書込権限
  - ECR読取権限

- **Playwright_User**: 開発者用ユーザー
  - ECS操作権限
  - ECR管理権限
  - CloudWatch Logs読取権限

### ネットワーク
- VPC内での実行（既存VPC使用）
- セキュリティグループで HTTPS (443) アウトバウンド許可
- NAT Gateway経由でインターネットアクセス（プライベートサブネット実行時）

### リソース保護
- すべてのAWS資源に `PlaywrightCloudExecuter` タグ付与
- 環境変数でAWSクレデンシャル管理
- ログ保持期間：30日（CloudWatch）

## リソース制限への対応

### 0.5vCPU / 1GB メモリ での最適化
- ブラウザ起動時に `--disable-dev-shm-usage` フラグ使用
- シングルスレッド処理
- ページロード待機時間：30秒以下
- メモリリーク防止：確実なページ・ブラウザクローズ

## スケーリング構想

## エラーハンドリング

### リトライメカニズム
- **ページ取得**: 最大3回までリトライ（1秒間隔）
- **S3保存**: エラー時は即座に失敗（リトライなし）

### ログレベル
- **DEBUG**: 詳細な処理フロー
- **INFO**: 処理の進捗・完了
- **WARN**: リトライ時
- **ERROR**: エラー発生時

## 日本語対応

### ロケール設定
- タイムゾーン: Asia/Tokyo
- 言語: ja_JP.UTF-8
- エンコーディング: UTF-8（全体統一）

### フォント対応
- Dockerfile に `fonts-noto-cjk` インストール
- 日本語を含むページのレンダリング対応

## 監視・アラート

### CloudWatch メトリクス
- タスク実行数
- 成功/失敗数
- 平均実行時間
- エラー率

### CloudWatch Logs
- アプリケーションログ
- エラースタックトレース
- S3保存結果

### アラート設定（推奨）
- 失敗時のSNS通知
- エラー率異常検知
